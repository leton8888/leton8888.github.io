例如:

nginx默认启动是在daemon模式下，所以在使用命令docker run -d nginx /usr/sbin/nginx时，容器启动nginx后会立刻退出，所以需要使用nginx的前台运行模式，需要在配置文件中加“daemon off”指令，或在启动时加“daemon off;”参数。注意off后面的分号不要忽略。

dockerfile 构建镜像

docker file 内容如下: dockerfile的名字必须为 Dockerfile 没有扩展名

docker build命令 -t 指定编译成镜像的名字 .是代表dockerfile在当前目录 注意要指定的是目录,不要指定名称,dockerfile文件名是固定的

docker file 后面加上一条 CMD [“/usr/sbin/nginx”,”-g”,”daemon off;”]

这样 docker run的时候,就等于自动执行了 nginx -g “daemon off;” 了,省得每次run的时候都要打这命令
所以CMD 和RUN 区别是 RUN是build的时候执行的,也就是build镜像的时候执行的东西
CMD是container运行是自动执行的命令,但是如果run container 时候运行了其他指令 CMD命令会被覆盖掉
例如:

container 运行的是 /bin/bash 而 nginx -g “daemon off;” 被覆盖掉了
如果不想被覆盖掉,就用entrypoint

ENTRYPOINT

要想覆盖ENTRYPOINT指令 请使用 docker run –entrypoint 覆盖

ADD 命令

如果想要把 nginx 默认的index.html换成Dockerfile同目录下的index.html文件 请使用ADD 命令

docker build 时候不使用缓存构建镜像 .代表当前shell的路径

比如RUN apt-get update 这条命令,我们想每次构建命令时候都update一下 这时候我们使用–no-cache命令
